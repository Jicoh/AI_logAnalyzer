{machine_info}

## 插件分析结果
{plugin_result}

## 筛选的日志内容
{log_content}

## 知识库内容
{knowledge_content}

## 用户请求
{user_prompt}

请根据上述信息，输出一个 JSON 对象，包含以下字段：

1. machine_info: 机器核心信息对象，必须包含以下字段（从插件结果中提取，若插件结果中无相关信息则填"未知"）：
   - serial_number: 序列号
   - product_name: 产品名称
   - board_id: 主板id
   - board_type: 主板类型
   - bmc_version: BMC版本
   - bios_version: BIOS版本
   - bme_version: BME版本
   - bmc_ip_address: BMC IP地址

2. summary: 问题摘要，包含 errors（严重问题数）、warnings（警告数）、info（提示数）
3. problems: 问题列表，每个问题包含 title、severity（error/warning/info）、description、source（可选）
4. solutions: 解决方案列表，每个方案包含 title、description、steps（步骤列表）、priority（high/medium/low）
5. log_snippets: 相关日志片段列表，每个片段包含 title（可选）、content
6. risk: 风险评估，包含 level（高/中/低）、description

直接输出 JSON 对象，不要包含代码块标记或其他解释内容。输出格式示例：
{{"machine_info": {{\"serial_number\": \"未知\", \"product_name\": \"未知\", \"board_id\": \"未知\", \"board_type\": \"未知\", \"bmc_version\": \"未知\", \"bios_version\": \"未知\", \"bme_version\": \"未知\", \"bmc_ip_address\": \"未知\"}}, "summary": {{\"errors\": 0, \"warnings\": 0, \"info\": 0}}, "problems": [], "solutions": [], "log_snippets": [], "risk": {{\"level\": \"低\", \"description\": \"\"}}}}
